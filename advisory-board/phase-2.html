Decisions Log
1. Dedicated Slack bot per conversational agent — July 23, 2026
Casey gets its own Slack app, token, and signing secret rather than persona-switching on the shared Northlight Claude bot. Keeps each agent's identity and routing clean; avoids ambiguity when both are active in the same channel.
2. app_mention (not message.channels) for Casey inbound — July 23, 2026
message.channels is not available in this workspace's Event Subscriptions UI. Switched to app_mention — architecturally cleaner anyway: Elizabeth explicitly @Casey to start a conversation rather than all channel messages being intercepted.
3. Cross-table memory uses soft gate, not separate API calls — July 23, 2026
Separate API calls per persona would be architecturally expensive and incompatible with the current multi-voice single-call design. Instruction-based scoping ("Jordan and Morgan only — don't reference this if you're not Jordan or Morgan") works well in practice, with the acknowledged limitation that it's not technically enforced.
4. Casey proactively offers ClickUp tasks, doesn't auto-create — July 23, 2026
Casey detects action items and asks "Want me to add this to Build Log Actions? Reply YES" — confirmation required before creating. Prevents unwanted task clutter; keeps Elizabeth in control of what gets tracked.
5. Build Log Actions as standalone ClickUp list — July 23, 2026
Casey can't infer which project a commit belongs to, so a per-project list approach would require manual context Elizabeth didn't want to provide. Standalone list (901715506061) is simpler and Casey can scan it without project context.

Technical Notes for Phase 3
Cross-table memory is soft-gated
All personas share one context window in a single API call. The instruction "Jordan and Morgan only — don't reference this if you're not Jordan or Morgan" works in practice but has no technical enforcement. If context bleed becomes a real problem before the 90-day check, it's a signal to separate the calls.
Casey's Slack bot requires CASEY_BOT_TOKEN + CASEY_SIGNING_SECRET in Railway
Both env vars are set. If the Railway service is redeployed from scratch, both must be re-added. The signing secret also needs to match what's in the Casey Slack App settings.
Casey scan recovery: Railway restart, not skill:casey-scan if wedged
The scan is stateless (reads ClickUp, writes to Slack, no transactions/locks). skill:casey-scan won't help if the process is wedged — use Railway → Restart. It reschedules cleanly on startup.
