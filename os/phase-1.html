Known Issues — Carried to Phase 2
Email archive not working
Archive button clickable but emails not actually archived from Gmail. Gmail API removeLabelIds call failing silently. Fixed in Phase 2.
Font colors too dark
Text barely readable on phone. #8A92A8 / #B0B8D8 colors insufficient on dark background. Brightened in Phase 2.
Dispatcher sends Slack as Elizabeth, not bot
Replies came back as self-DMs. Required Slack App (bot identity) before clarification loop could work properly. Resolved by Slack App build.
Not mobile responsive
Desktop-only at Phase 1 close. Planned for Phase 3. Fixed in Phase 2 as part of tab restructure.
OAuth defaulting to personal Gmail
Dashboard defaulting to elizabethlemoine33@gmail.com instead of elizabeth@bynorthlight.ca. Addressed in Phase 2 dual-account setup.
Dispatcher clarify shows placeholder
Slack message sent "[your question]" instead of actual CLARIFY text from task. Fixed in Phase 2.
Decisions Locked in Phase 1
ClickUp stays as the task database — no replacement
The goal shifted from "find a better tool" to "build the layer that makes the tools we have work together."
Claude has a real ClickUp account: claude@bynorthlight.ca
Enables tasks to be assigned to Claude, Claude to mark things done, and notifications to flow correctly.
All Northlight work files live in bynorthlight.ca Google Drive
Personal Gmail Drive is legacy — files to be migrated over time. Keeps work identity and personal identity separate.
Skill tags use format skill:[name] as a ClickUp tag
Tells the dispatcher which agent to load. Tags defined: aurora-brief, brand, clickup, vault, tableready, career, web, strategic.
Clarification flag: CLARIFY: [question] at top of task description
Two-hour reply window before Claude proceeds with best assumption and flags it in output.
Claude needs a Slack App (not a user account)
Proper bot identity so Elizabeth can reply naturally in a real DM thread. Clarification loop cannot work without this.
Google OAuth only — no passwords stored anywhere
Two allowlisted accounts: elizabeth@bynorthlight.ca (primary) and elizabethlemoine33@gmail.com (backup). Any other Google account rejected even if OAuth succeeds.
CRM does not live in the OS web app
Client data on a public web layer without serious security architecture is a privacy risk. Future home is ClickUp. Revisit when Northlight has paying clients.

Pipelines Defined in Phase 1
Pipeline 1 — Aurora Brief (Newsletter)
research → strategic → draft → aurora-brief review → draft (revisions) → polish → file-output → Elizabeth publishes via Kit. ~4–6 hours.
Pipeline 2 — Vault / Web Copy
draft → web (standards review) → polish → file-output. Brand + Vault reference skills always loaded alongside.
Pipeline 3 — Career / Job Applications
career → docx (format) → file-output → Slack link. Standalone — not integrated with other pipelines.
Pipeline 4 — Daily Dispatcher Loop
6am: query ClickUp → check CLARIFY flag → load skill → execute → file-output → update ClickUp status → Slack summary to Elizabeth.
Pipeline 5 — Morning Dashboard Load
Dashboard queries ClickUp (done tasks), Drive (overnight files), Calendar (today's blocks) → displays Overnight by Claude, Critical Now, Today's Calendar, Project Health.
Pipeline 6 — Expense Extraction & Approval
Gmail label → expense-extract → Slack #expenses → Elizabeth approves/edits/rejects → expense-approve updates Google Sheet → monthly expense-summary.
Pipeline 7 — Blue Bear Client Advisory
Weekly Friday: blue-bear-review queries ClickUp + Calendar + asks Elizabeth for learnings → Drive doc + Slack post + dashboard panel.
Pipeline 8 — Project Health
Daily: project-health counts tasks by status → Green/Yellow/Red → dashboard panel. Used by blue-bear-review for client health calculation.
Pipeline 9 — Email Inbox Flagging
Morning: email-inbox filters Gmail (unread 24hr+, no subscriptions) → dashboard Inbox panel → quick reply / defer / archive actions.
