Finance Sheet Architecture (Defined & Built)
Sheet 1 — Expense Log
Date · Vendor · Amount · Category · Project · Notes. Auto-populated from Gmail label approval workflow. 15 expense categories defined.
Sheet 2 — P&L Summary
Revenue: Product Sales (Vault + TableReady), Services, Sales, Total. Expenses: grouped by type. Net Income / Loss. By-project breakdown (Northlight, Vault, TableReady).
Sheet 3 — Charts
Monthly spending trend (line), spending by category (bar), spending by project (pie), "In the Red" status headline. All auto-updating from Expense Log.
Sheet 4 — Category Reference
All 15 expense categories for dispatcher dropdown + validation. Export-ready for accounting software.
Decisions Locked in Phase 2
Tab-based UI — not single-scroll
Too many panels for one scroll surface. Tabs give each workflow its own space. Design navigation structure before adding panels, not after.
Slack-review-first before expense logging
AI-guessed categories are right most of the time, not always. Human confirmation at approval is faster than correcting logged errors. Sheet stays clean.
Google Sheet as financial source of truth
Already export-ready for accounting software. Charts auto-update. Shareable with accountant without granting dashboard access.
15 expense categories — structured for tax readiness
Computer Hosting/Software/Hardware/Internet, Bank Charges, Office Supplies, Meals & Entertainment, Utilities ×4, Professional Fees, Owner Draw, Uncategorized.
Parking Lot wires to existing Slack channel
Channel (#parking-lot-of-ideas) already had ideas in it — no migration needed. Wire the existing source rather than create a new one.
Deferred to Phase 3
localStorage → Drive-backed persistence for Wins + Learning
Wins Board and Learning Log used localStorage in Phase 1. Data lost on browser clear or device switch. Drive persistence was the fix — built in Phase 3.
Mobile responsiveness
Tab UI improved things but full mobile responsiveness was planned for Phase 3. Dashboard still primarily desktop-optimized at Phase 2 close.
Goals dashboard
Annual + quarterly goals (ClickUp Goals API or Drive JSON fallback). Designed and specced in Phase 2 planning; not built until later.
Revenue tracking
Expense side of P&L fully built. Revenue side (northlight-revenue Gmail label → same workflow) designed in Phase 2, wired in Phase 3.
