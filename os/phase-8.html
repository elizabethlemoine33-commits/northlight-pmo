Decisions Made — July 18, 2026
Skeleton key scope: path-check required in requireAuth, not server.js pre-middleware
The dead pre-auth middleware in server.js (lines 87–96) called next() in both branches — useless. Real fix was in requireAuth: require both key match AND path match. server.js dead code left alone as harmless.
Integration diagnostics duplicated to Maintenance, not moved
Settings is for configuration; Maintenance is for health checks. They serve different mental models even though the data is the same. Both cards call /api/health — no extra backend work.
Dispatcher run log capped at 20 entries in Drive
Drive JSON is not a database — unbounded appends would bloat the key over time. 20 entries is enough for pattern detection without storage risk. UI shows 8.
Console format standardization: 683 statements, most already prefixed
Package B estimated 268 statements; actual count was 683. Investigation showed all major files (dispatcher, auth, slack-events, analytics, archive, dedup, revenue) already use consistent [MODULE] prefixes. Only auth.js reconnect lines were missing timestamps — fixed in place. No broad sweep needed.
Error translation: pipeline-utils already covers user-facing errors
firePipelineSkill wraps every auto-fired skill with formatErrorForSlack. The remaining ~70 catch blocks are internal debug logs — wiring translateError into them would add noise without user benefit. Closed the ClickUp task as resolved-by-design.
Failed processes surfaced via existing social queue status: 'failed' field
Social posts already write status: 'failed' + error + failedAt back to the Drive queue on publish failure. No separate failure log needed — QueueMonitorCard reads and surfaces these inline.
googleapis CVE is a separate track — not a Package
It's a dependency upgrade, not a UI or observability task. HIGH priority but breaking change — warrants its own scoped session. ClickUp 86e2bp3gh.
