Buffer scheduled times must be converted to Halifax local, never treated as UTC
The Railway server runs in UTC, so new Date("...5:00 PM") silently parsed as 5:00 PM UTC = 2:00 PM Halifax — already past by the time Buffer received it, causing a real publish failure on the first live test. Fixed by measuring Halifax's actual UTC offset at parse time (handles DST automatically) rather than any fixed offset.
