The Standing PMO system was built July 3, 2026 — repo, GitHub Pages, dispatcher skill, scheduler, and Slack listener all wired. Phases 1 and 2 fix the data loop and the ClickUp structure so the rest of the architecture plan can build on a solid foundation.

Decisions Log
1. STANDING-PMO-TRIGGER check moves before the message guard
The trigger is posted by the bot — it can never pass an Elizabeth-only filter. Minimal correct fix. · Phase 1
2. ClickUp write-back embedded in phase-dashboard skill
By the time a phase dashboard generates, ClickUp is already stale. Embedding it in Step 2b ensures it runs automatically. · Phase 1
3. CLAUDE.md in northlight-pmo for standing PMO behaviour
Applies automatically to any session in PMO context — no session prompt needed. · Phase 1
4. One folder, one list per project in ClickUp
Multiple lists per project made registry-driven sweeps impossible. One list per project = one ID per project = clean. · Phase 2
5. Social Media excluded from Project Backlogs
Tasks must live in the Marketing folder by pipeline (Threads, LinkedIn, Blog, Aurora Brief) for content planning. Stays null in registry. · Phase 2
6. Phase 1+2 dashboards combined; Phase 3+4 combined
Phases 1 and 2 are both foundation work done in one session. Phases 3 and 4 are two sides of the same swap (retire old, build new). · 2026-07-16
