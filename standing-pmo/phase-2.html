Decisions Log
1. Status-based task filtering (not name prefix or tags)
Lowest friction — no naming convention required, no tagging process. Backlog = not in scope; ready/in-progress = active; complete after phase_started = done this phase. · 2026-07-16
2. Slide-out drawer (not popup or separate page)
Keeps user in OS context. Popup loses context on dismiss; separate page loses project list. Drawer is the right containment for drill-down detail. · 2026-07-16
3. Health status deferred to Phase 5
Need to define the rubric before building: pre-build dormant ≠ unhealthy; building with no in-progress may be stalled; live with blockers = unhealthy. Can't implement without the rubric. ClickUp task 86e2cjp77 created. · 2026-07-16
4. Old empty ClickUp lists — deleted (not archived)
Elizabeth deleted them. Decision closed — no archive needed, no harm. · 2026-07-16
5. getLatestDashboardUrl() via GitHub API — no hardcoded filenames
Dashboard links on OS project cards must survive phase increments. GitHub API returns the folder contents → find highest phase-N.html → link to it. No manual update needed on each phase commit. · 2026-07-16
6. /phase-dashboard and /new-project skills auto-update registry.json
Phase label on OS card must be accurate without manual registry edits. Embedding Step 7b in phase-dashboard and equivalent in new-project ensures the phase fields are always current after a skill run. · 2026-07-16
