Decisions Log
1. Slack OS link uses DASHBOARD_URL env var (not hardcoded)
Railway URL could change on redeploy if the service is recreated. Using the existing DASHBOARD_URL env var means no code change needed if the URL ever changes. · 2026-07-16
2. Retired index.html has no link to the OS
The OS is internal — linking to it from a public GitHub Pages URL would expose the Railway URL unnecessarily. The retired page just says the view moved internally. · 2026-07-16
3. phases[] is newest-first and tracks closed phases only
Current phase is already in current_phase fields. phases[] accumulates history — closed entries only, newest at front. This enables "how long in each phase" analysis without parsing markdown. · 2026-07-16
4. Filter/sort is client-side on projects.json — no server change
All project data is already in projects.json. Adding a sort order field to registry.json would require a dispatcher change and STANDING-PMO-TRIGGER run to update. Client-side sort is instant and requires no infrastructure. · 2026-07-16
5. Health status rubric deferred to end of Phase 5
A wrong rubric produces wrong health signals across all 9 projects. Pre-build dormant ≠ unhealthy; building with 0 in-progress may be stalled or just planned. The rubric decision must come before the implementation. · 2026-07-16
