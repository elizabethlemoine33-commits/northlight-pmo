Integrations Being Built
Google OAuth — Desktop Loopback
Temporary HTTP server on a random port intercepts the redirect from Google after browser auth. No express needed — pure Node http module. Tokens (access + refresh) encrypted with Electron safeStorage (same pattern as Groq key). Auto-refresh checks expiry before every API call.
Google Drive + Docs Push
App copies a master template Google Doc into the target Drive folder, names it "{Client} — {Project} — SOW" (or Meeting Notes). Fills all {{tag}} placeholders via Docs API batchUpdate + replaceAllText. Returns a clickable Doc URL stored in the session record.
ClickUp Task Creation
Discover mode only: approved clickupTasks from the analysis are created in the new client actions list. Priority mapped from urgent/high/normal/low. ClickUp API key stored via safeStorage. List ID configurable in Settings. Task IDs saved back to session record.
Google Calendar Events
Approved extractedDates from the analysis are pushed to Google Calendar primary. Each date becomes an event — title from description, duration defaults to 1 hour for timed events, all-day for date-only isoDate values. Event IDs saved to session record.

Architecture Decisions
OAuth: desktop loopback via temp HTTP server
No Electron custom protocol needed. A random-port localhost server intercepts the redirect, exchanges the code, then shuts down. Same pattern used by Google's own desktop quickstart examples.
Token storage: safeStorage (same as Groq key)
Refresh token encrypted with Windows DPAPI via Electron safeStorage. Not in plain JSON. Access token refreshed in-memory on demand, not stored.
Drive push: copy-from-master-template, not build-from-scratch
Elizabeth imports the two HTML templates into Drive as Docs once. Each session copies the template into the target folder and fills tags. Preserves Northlight formatting, no layout code needed in the app.
ClickUp: new client-actions list, separate from app backlog
List 901714819070 is now reserved for app development tasks. Client tasks from Discover calls go into a new dedicated list. List ID hardcoded in Settings — no per-session picker needed.
Push results persisted in session JSON
docUrl, taskIds[], eventIds[] written back to the session record after each push. Re-opening the session shows "already pushed" state and the Doc link — no need to re-push.
Replit source: port, don't rewrite
google-oauth.ts and integrations.ts (lines 145–513) contain working implementations of all four integrations. Phase 3 adapts them — remove framework deps (express, drizzle), use native fetch, replace token DB with safeStorage.

Drive Push — Template Tag Flow
sowDraft (Discover mode) → Proposal template copy
Phase 2 locked in the output schema. The push is a direct mechanical loop:

Object.entries(sowDraft).map(([tag, val]) => ({ replaceAllText: { containsText: { text: "{{" + tag + "}}" }, replaceText: val ?? '' } }))
Plus metadata fills: {{client_name}}, {{project_name}}, {{date}}, {{prepared_by}} from session record. Fields the LLM left null are replaced with empty string — user edits in Drive.
meetingNotesDraft (Progress/Notes mode) → Meeting Notes template copy
Same pattern. meetingNotesDraft keys match the Northlight_MeetingNotes_APITemplate {{tags}} exactly. Metadata: meeting_date from session timestamp, prepared_by from settings or default "Elizabeth Lemoine".
