Decisions Made
Electron over Tauri/Rust fork of Meetily
No Rust toolchain or MSVC on machine. Node.js v24 already installed. Lower setup friction — can build today.
Groq Whisper over local Whisper.cpp
Intel Iris Plus integrated GPU makes local Whisper too slow (~10–15 min for a 1-hr call). Groq transcribes the same in ~30–60s for $0.111/hr.
Groq LLM over Ollama or Claude for analysis
Same GPU constraint rules out Ollama. Groq Llama 3.3 70B is ~$0.03/call — much cheaper than Claude Sonnet. Analysis is Phase 2.
VB-Audio Virtual Cable for system audio
Standard Windows approach. Free. Already installed. Simpler than WASAPI native module in Node.js.
Phase 1 = recording + transcription only
Prove the audio pipeline works before adding any workflow complexity. Analysis, ClickUp, Drive are Phases 2–3.
Open Issues / Watch Items
System audio (VB-Audio Cable) not yet confirmed
Mic capture working. VB-Cable requires Windows default playback device to be set to VB-Cable during calls. Workaround: taskbar audio switcher (3 seconds). In-app switcher added to Phase 2 backlog.
Groq free tier limits to monitor
20 req/min, 100/day, 28.8k audio-sec/hour. At 5 calls/week this is fine. Auto-upgrades to paid if exceeded — note for Elizabeth.
safeStorage DPAPI key loss on reformat
Electron safeStorage on Windows uses DPAPI — keys are tied to the Windows user account. Warn in Settings UI: "If you reinstall Windows, re-enter your API key."
Audio file size for very long calls
Groq Whisper limit is 25MB. A 1-hr call at 16kbps opus webm ≈ 7MB — fine. Only an issue for 3+ hour calls. Monitor but don't solve in Phase 1.
