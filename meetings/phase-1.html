Bugs Found + Fixed (6 total)
#1
CSP meta tag blocked style-loader CSS injection — app launched with no theme, all screens visible simultaneously
Fix: Removed CSP meta tag from index.html entirely. style-loader injects CSS as inline style tags, which the restrictive CSP blocked.
#2
TypeError: fetch is not a function — node-fetch incompatible with webpack bundling in Electron main process
Fix: Removed node-fetch import. Switched to Node 24 built-in global fetch (available natively, no package needed).
#3
form-data package incompatible with native fetch — getHeaders() method specific to node-fetch
Fix: Replaced form-data with native Blob + FormData globals. Audio buffer wrapped in Blob, appended to native FormData.
#4
Realtek Audio Console had laptop speakers muted at driver level — audio reached Voicemeeter A1 bus but produced no sound
Fix: Opened Realtek Audio Console → Playback Devices → unchecked Mute on Speakers. Diagnosed via computer-use screenshot showing signal on A1 fader meter but no output.
#5
Voicemeeter Out B1 device label not in system audio detection — app couldn't find the capture device by label
Fix: Added "voicemeeter out b1" and "voicemeeter out b" to label matching in renderer.js getUserMedia device detection.
#6
Mic gain imbalance — system audio (YouTube/call app) dominated the merged stream; mic inaudible in transcript
Fix: Added GainNode per stream. Mic: 2.0×, system: 1.5×. Calibrated against real conversational audio to ensure both voices appear in transcript.
Key Decisions
Electron over Tauri/Rust
No Rust toolchain or MSVC on machine. Node.js v24 already installed. Could build same day vs 45-min prerequisite install for Tauri.
Groq Whisper over local Whisper.cpp
Intel Iris Plus integrated GPU makes local inference ~10–15 min per 1-hr call. Groq transcribes the same in 30–60s for $0.111/hr.
JSON file storage over SQLite
better-sqlite3 requires MSVC native compiler, not installed. JSON at %APPDATA% is simpler, sufficient for Phase 1.
Voicemeeter Banana over VB-Cable-only
VB-Cable alone silenced the speakers during recording (no audio monitor). Voicemeeter routes audio to both speakers (A1) and capture device (B1) simultaneously.
Phase 1 = record + transcribe only
Validate the audio pipeline before adding LLM analysis, ClickUp, Drive, or Calendar. Correct sequencing — each phase builds on confirmed foundation.
