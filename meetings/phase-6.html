Decisions Log
#DecisionWhyDate
1 Audio check on pre-call open, not as a Settings page item The check is most useful immediately before a recording, not buried in Settings. Surfacing it on the New Session screen means it runs exactly when it matters and doesn't require the user to remember to check. 2026-07-13
2 Fix button calls PowerShell (AudioDeviceCmdlets) rather than opening Settings Opening Windows Settings requires the user to navigate three levels and make the change manually. The PowerShell call fixes it in one click with no navigation — faster and less error-prone. 2026-07-13
3 Hardcode AudioDeviceCmdlets DLL path rather than module name Electron spawns PowerShell without the user profile, so the CurrentUser module path isn't in PSModulePath. Importing by full DLL path bypasses the profile entirely and works reliably in headless sessions. 2026-07-13
4 Meet speaker → CABLE Input (in Meet settings) rather than system default Setting system default to CABLE Input breaks all Chrome audio between calls. Google Meet persists the speaker setting per Chrome profile — routing only Meet through the cable leaves everything else on Realtek with no toggling needed. 2026-07-13
