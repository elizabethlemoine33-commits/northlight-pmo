Template-Keyed JSON — Design Decision
sowDraft output is a structured object, not a freeform string
The original Replit app returned sowDraft as a single markdown string. Phase 2 changes this to a JSON object with keys matching the Northlight Proposal API Template exactly:

{{problem_statement}} {{in_scope_1}} {{deliverable_1}} {{approach_summary}} {{phase1_focus}} {{phase1_timing}} {{project_fee}} {{payment_terms}} {{next_steps_text}} + 14 more

Phase 3 Drive push: Object.entries(sowDraft).map(([tag, val]) => replaceAllText request) — no parsing layer needed.
meetingNotesDraft follows the same pattern
Progress and Notes modes output a meetingNotesDraft object keyed to the Northlight Meeting Notes API Template:

{{meeting_type}} {{key_point_1}} {{decision_1}} {{decision_1_owner}} {{action_1_owner}} {{action_1_item}} {{action_1_due}} {{next_meeting_date}} + 12 more

client_name, project_name, date, and prepared_by are filled from session metadata — not by the LLM.
Key Decisions
Groq Llama 3.3 70B for analysis
Same API key as Whisper. ~$0.03/call for analysis. Temperature 0.3 for consistent structured output. Total per-call cost: ~$0.14 for a 1-hour call.
Post-call feedback as a separate transcription pass
Verbal note recorded after stopping, transcribed in parallel with the main audio, injected into the LLM prompt as HIGH PRIORITY context that overrides or clarifies the transcript. Adds no latency to the pipeline.
approved flag on every array item
Each action item, task, decision, commitment, etc. carries an approved field (default false from LLM, toggled in review screen). Phase 3 push only sends approved items. Unapproved items remain in session JSON for reference.
Template keys baked into prompts, not mapped post-hoc
The LLM is instructed to output keys matching the Google Docs template tags directly. Earlier approach (freeform SOW string → manual extraction) was fragile. This approach makes Phase 3 a mechanical loop with no ambiguity.
Recording-screen prompts are static, not AI-generated
Cheat sheet items are hardcoded per mode, not generated by the LLM. Faster, no cost, available before the call starts, and predictable. Covers the exact fields the analysis needs to produce good output.
